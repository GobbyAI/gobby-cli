# crates/gcode

Parent: [[code/modules/crates|crates]]

## Overview

The `crates/gcode` module implements gcode, a Postgres-backed code intelligence CLI and library for indexing, searching, and analyzing source code across many languages.

It is organized into three parts:

- **src** — The implementation root containing the full toolchain: language-aware parsing and symbol/import/call extraction (with C++ clangd integration), file discovery and classification, multi-language import resolution, freshness and project-lock management, and identity/config/database bootstrapping. Search spans symbol, text, content (BM25), grep, and semantic/vector retrieval, alongside a FalkorDB-backed code graph supporting callers, usages, imports, blast-radius, lifecycle sync, and project graph reports. Higher-level features include hierarchical "codewiki" documentation generation (with AI prompting, citation grounding, and ownership/blame attribution), Qdrant vector lifecycle management, embeddings doctor diagnostics, standalone Postgres setup, and skill installation.

- **contract** — A single machine-readable JSON artifact (`gcode.contract.json`) defining the versioned CLI surface: global flags, per-command definitions, identity keys, JSON output keys, and error codes, serving as the source of truth for parsing, validation, and tooling integration.

- **assets** — Static JSON data cataloging well-known dependency and import roots (Elixir and Ruby) to help the indexer recognize standard-library and third-party package boundaries during import resolution, plus a bundled SKILL.md.

A core design principle, enforced by tests, is that the library API remains CLI-independent.
