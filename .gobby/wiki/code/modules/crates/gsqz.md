# crates/gsqz

Parent: [[code/modules/crates|crates]]

## Overview

The `gsqz` crate is a configurable command-output compression tool that shrinks verbose shell output while preserving meaningful content. It is driven by a YAML configuration (`config.yaml`) defining global settings (minimum output length, max compressed lines, empty-output behavior), a set of command-matched pipelines, and a fallback path. Each pipeline matches commands via regex and applies an ordered sequence of compression steps.

The core engine (`src`) compiles pipelines and runs a `Compressor` that splits compound commands, applies built-in and configured exclusions, selects the matching pipeline by command basename, and produces a `CompressionResult` tracking savings percentage and passthrough status. Compression steps are modular: line filtering, mode-based grouping (git status/diff, pytest and generic test failures, lint-by-rule, by-extension/directory/file, errors/warnings), deduplication of near-identical lines, regex replacement, head/tail truncation (whole or per-section), early-exit `match_output` rules, and multi-level prose compression that protects code blocks, URLs, paths, and frontmatter.

A CLI front end supports both input and output modes, optional daemon-based config fetching, and savings reporting. The module ships a comprehensive test suite covering command splitting, exclusion logic, configuration loading/serialization, and every compression step.
