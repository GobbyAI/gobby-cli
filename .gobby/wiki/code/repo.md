# Repository Overview

## Overview

Gobby is a Rust-based developer toolchain organized as a Cargo workspace. Its core lives in the `crates` directory, which groups multiple standalone CLI/library components, each with a focused role across the tool suite (gcode, ghook, gloc, gsqz, gwiki, gcore).

The `docs` directory provides comprehensive reference material in Markdown, spanning interface contracts (gcode, gwiki, research workflows, a shared graph schema), user and developer guides, AI configuration, daemon/install specs, codewiki, release processes, active and completed planning documents, and research spikes. The `scripts` directory holds shell utilities, currently a single `verify.sh` verification script.

Root-level files include standard project metadata and documentation (README, CHANGELOG, CONTRIBUTING, LICENSE, AGENTS/CLAUDE guidance), Cargo workspace configuration (Cargo.toml, Cargo.lock), Rust toolchain pinning, and supplementary analysis assets. [1] [17] [27] [490] [505]

## Modules

- [[code/modules/crates|crates]] - The `crates` workspace groups the Gobby toolchain's component crates, each a standalone CLI/library with a focused role:

- **gcode** — A Postgres-backed code intelligence CLI and library for indexing, searching, and analyzing source code across many languages (graph, vector, and BM25 search; symbol resolution; codewiki generation).
- **gcore** — Shared foundational layer providing config resolution, service clients (Postgres, FalkorDB, Qdrant), AI routing, graph analytics, schema/setup, and secret handling consumed by the other crates.
- **ghook** — A hook-dispatch CLI bridging agent tooling (Claude, Codex, Droid, Gemini) to the gobby daemon, wrapping hook events in versioned envelopes and translating daemon responses into hook actions.
- **gloc** — A CLI that launches AI client binaries against configurable LLM backends (including local Ollama), resolving backends, clients, models, and environment from layered YAML config.
- **gsqz** — A configurable command-output compression tool that shrinks verbose shell output via regex-matched pipelines defined in YAML.
