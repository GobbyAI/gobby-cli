## crates/gcode/src/index/walker

The `walker` module is the entry point for deciding **which files should be indexed** and **how**. It combines filesystem traversal, gitignore awareness, and a multi-stage filtering pipeline to produce two sorted lists of absolute paths: AST-eligible candidates (files whose language is detected and whose size is within policy) and content-only candidates (files that should be chunked but not parsed into a symbol graph). The public surface is intentionally narrow — callers reach in through `discover_files` / `discover_files_with_options` for bulk project walks and through `classify_file` / `classify_explicit_file_with_options` for single-path queries discovery.rs:11-17 classification.rs:14-54.

File classification proceeds through a layered rejection pipeline. `is_safe_text_file` (classification.rs) gates on size (`MAX_FILE_SIZE`) and security rules imported from `crate::index::security`. Surviving files are then tested against `is_generated_wiki_metadata` and `is_generated_js_bundle` — both of which return early with `None` — before being routed to `FileClassification::Ast` or `FileClassification::ContentOnly`. The JS-bundle detector in `generated.rs` first scans the initial 64 KB for known comment markers (`generated by`, `@generated`, etc.) and, for files above `MINIFIED_JS_MIN_BYTES` (128 KB), applies a line-length heuristic that counts lines and average line bytes to detect minification generated.rs:1-93. Oversized data-language files (JSON/YAML exceeding `MAX_DATA_LANGUAGE_AST_SIZE`) are demoted to content-only to avoid bloating graph/vector/FTS projections (classification.rs:36-48, referencing gobby-cli #678).

The hidden-path subsystem (`hidden.rs`) handles files that a normal gitignore-respecting walker would skip. `HiddenPathAllowlist::load` seeds itself with four built-in glob patterns and then merges any project-specific additions from `.gobby/gcode.json` hidden.rs:4-26. The allowlist expands `**` at depth zero, resolves patterns to absolute globs, and calls `glob::glob` to materialise matching paths; only files confirmed by `is_hidden_path` are admitted hidden.rs:37-54. The discovery loop in `discovery.rs` calls `HiddenPathAllowlist::load` after the main walker pass and feeds each result through the shared `push_classified_file` helper, which canonicalises the path and uses a `BTreeSet` to prevent duplicates discovery.rs:56-84.
